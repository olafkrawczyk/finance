<objective>
Build the domain layer: TypeScript entities, Zod v4 validation schemas, and the use-case functions that implement all Phase 1 business logic (transaction create with atomic PGMQ enqueue, paginated listing, monthly summary computation, and opening-balance CRUD). Output: entities.ts, schemas/ledger.ts (Zod v4), use-cases.ts, and filled-in DB + schema tests.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | SQL injection via use-case params | mitigate | All queries use postgres.js tagged template literals; conditional fragments use nested `sql`` not string concat; no sql.unsafe in use-cases.ts |
| T-02-02 | Tampering | numeric precision loss on amounts | mitigate | NUMERIC kept as string end-to-end; parseFloat only for summary arithmetic, output re-stringified via toFixed |
| T-02-03 | Denial of Service | lost background work on partial failure | mitigate | createTransaction wraps INSERT + pgmq.send in a single sql.begin transaction (atomic) |
| T-02-04 | Tampering | inflated summary via transfers | mitigate | summary query filters `WHERE t.type != 'transfer'` (REQ-1.4) |

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client -> API | Untrusted JSON bodies and query params cross into Hono routes |
| API -> use-cases | Only Zod-validated data is forwarded to the domain layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | POST/PUT request bodies | mitigate | @hono/zod-validator validates every body against Zod v4 schemas; failure hook returns 400 envelope before any DB write (ASVS V5 Input Validation) |
| T-03-02 | Information Disclosure | error responses leaking internals | mitigate | Catch blocks return a generic `{ message }`; only 'immutable'/'duplicate' strings drive status codes, raw stack traces are not returned |
| T-03-03 | Tampering | malformed query params on GET /transactions | mitigate | ListTransactionsQuerySchema coerces and bounds page/per_page (max 500) and validates uuid/date filters |
| T-03-04 | Spoofing | unauthenticated access | accept | No auth in Phase 1 (V2/V4 out of scope per RESEARCH Security Domain); Better Auth lands in Phase 2 |
</threat_model>

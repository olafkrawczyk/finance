## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Spoofing | POST /accounts | mitigate | userId extracted from c.get('user').id (session), never from request body — matches Phase 7 SCOPE-06 pattern |
| T-11-02 | Repudiation | DELETE guard | mitigate | SELECT before DELETE pattern — count transactions referencing the account, return 409 with count before any mutation |
| T-11-03 | Information Disclosure | PUT/DELETE user_id filter | mitigate | WHERE id = ${id} AND user_id = ${userId} in every mutation — User B gets empty result (404) for User A's account |
| T-11-04 | Tampering | Name uniqueness | mitigate | UNIQUE(user_id, name) constraint at DB level — second INSERT with same name for same user throws unique violation |
</threat_model>

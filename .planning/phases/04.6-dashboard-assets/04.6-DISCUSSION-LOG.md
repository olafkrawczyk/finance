## Cash Asset Handling

| Option | Description | Selected |
|--------|-------------|----------|
| Manual Asset | Track cash manually as a customizable asset line item (alongside Obligacje, ETF, etc.), keeping it simple and fully editable. | ✓ |
| Sync with ledger | Sync the cash asset automatically with the sum of all bank account balances in the app, and allow adding additional manual cash. | |
| Dynamic display | Do not have a specific Cash asset; simply display the current bank balance dynamically from the database and add it to the manual assets. | |

**User's choice:** Track cash manually as a customizable asset line item (alongside Obligacje, ETF, etc.), keeping it simple and fully editable.

## Historical Asset Value Tracking

| Option | Description | Selected |
|--------|-------------|----------|
| Snapshot only | Keep it simple: only track the current snapshot values of assets (no historical charts). | ✓ |
| History log | Keep a history log: save a snapshot of asset values each month so we can show a "Net Value Over Time" chart in the future. | |

**User's choice:** Keep it simple: only track the current snapshot values of assets (no historical charts).

## Current Month Fallback

| Option | Description | Selected |
|--------|-------------|----------|
| Latest month fallback | Fall back to showing the most recent month in the database that contains data (with a label indicating which month it is). | ✓ |
| Show calendar month | Show the current calendar month anyway, displaying 0.00 PLN for expenses/income/savings if there are no transactions yet. | |

**User's choice:** Fall back to showing the most recent month in the database that contains data (with a label indicating which month it is).

## Y-Axis Formatting

| Option | Description | Selected |
|--------|-------------|----------|
| Format without decimals | Format values with spaces for thousands but omit decimal places (e.g. "12 000" instead of "12 000,00") to keep the axis clean. | ✓ |
| Format with decimals | Format values with spaces for thousands and include decimal places (e.g. "12 000,00"), matching the tables exactly. | |

**User's choice:** Format values with spaces for thousands but omit decimal places (e.g. "12 000" instead of "12 000,00") to keep the axis clean.

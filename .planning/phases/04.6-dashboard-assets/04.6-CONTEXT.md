**Gathered:** 2026-06-07
**Status:** Ready for planning

<domain>
## Phase Boundary

</domain>

<decisions>
## Implementation Decisions

### 1. Asset Management (Total Net Value Tile)
- **D-01:** Add a new table `assets` to track custom assets manually. Fields: `id`, `name`, `value`, `created_at`, `updated_at`.
- **D-02:** Build Hono CRUD routes: `GET /assets`, `POST /assets`, `PUT /assets/:id`, `DELETE /assets/:id`. Protect with `requireAuth` middleware.
- **D-03:** Store asset values as `NUMERIC(19, 4)` and ensure values are non-negative.
- **D-04:** Assets are purely tracked as current snapshot values. No history tracking or historical charts are required for assets in this phase.
- **D-05:** "Cash" is treated as a manual asset line item alongside others (obligacje, ETF, silver) to keep the asset model simple, uniform, and fully editable.
- **D-06:** Build a new frontend page `/assets` (AssetsPage) for managing assets (adding new ones, editing values/names, deleting them).
- **D-07:** Add navigation link for "Aktywa" in the header layout.
- **D-08:** Add a "Total Net Value" dashboard tile displaying the sum of all asset values, with a link/button to navigate to `/assets`.

### 2. Current Month Summary Tile
- **D-09:** Add a dashboard tile displaying three numbers for the current month: **Przychody** (Income), **Wydatki** (Expenses), and **Zaoszczędzone** (Savings = Income - Expenses).
- **D-10:** Extract the current month's numbers from the existing `/transactions/summary` data.
- **D-11:** If the current calendar month has no transaction data yet, fallback to the most recent month containing data available in the dataset, with a label clearly indicating which month is displayed.

### 3. Chart Number Formatting
- **D-12:** Format all chart tooltips and Y-axis labels using the `pl-PL` locale (space as thousand separator, comma as decimal separator).
- **D-13:** Chart tooltips will show full precision (e.g. `12 345,67`).
- **D-14:** Chart Y-Axis tick labels will format whole numbers without decimals (e.g. `12 000` instead of `12 000,00`) to maintain a clean layout.

</decisions>

# Phase 4.5 Plan 1: Backend Schema Migration + API Routes + Use-Cases + Tests for Transaction CRUD

**Removed transaction immutability: schema migration (triggers, updated_at), three new API endpoints (GET/PUT/DELETE /transactions/:id), use-case functions, and updated test suite**

## Accomplishments

- Schema migration applied to Postgres: `block_immutable_change()` now returns NEW unconditionally, `block_delete()` function and trigger removed, `updated_at TIMESTAMPTZ` column added, `update_transaction_timestamp()` BEFORE UPDATE trigger created
- `Transaction` entity interface extended with `updated_at: string | null`
- `UpdateTransactionSchema` (reuses `CreateTransactionSchema`) and `UpdateTransactionInput` type added
- Three new use-case functions: `getTransaction()`, `updateTransaction()`, `deleteTransaction()`
- Three new API routes: `GET /:id`, `PUT /:id`, `DELETE /:id` on `ledgerRoutes` — all with `requireAuth`, UUID validation, Zod body validation (PUT), and envelope response format
- `deleteTransaction()` runs atomically: clears `import_hash`, removes ID from `insights.linked_transaction_ids` via `array_remove`, then hard deletes
- Test suite updated: old immutability test rewritten, new tests for `getTransaction()`, `updateTransaction()`, and `deleteTransaction()` added
- api.test.ts immutability test updated to match new trigger behavior

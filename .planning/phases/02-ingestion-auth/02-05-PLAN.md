| User file → upload | Client-side file type/size validation (defense-in-depth; server is authoritative) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-13 | Spoofing | cross-origin auth cookie | mitigate | Backend CORS (set in 02-01) restricts origin to FRONTEND_URL with credentials; `credentials: 'include'` only sends cookie to the configured origin (ASVS V3) |
| T-2-14 | Tampering | client-side file validation | accept | Client checks .csv + 10MB for UX only; server-side validation in 02-03/02-04 is authoritative (defense-in-depth) |
| T-2-15 | Information Disclosure | error display | mitigate | Error list shows server-provided messages only; no secrets rendered; descriptions are plain text (no formula/HTML injection in React text nodes) (ASVS V13) |
| T-2-SC | Tampering | npm installs (react/vite/tailwind toolchain) | mitigate | [ASSUMED] packages gated by a blocking-human legitimacy checkpoint (Task 0) verifying registry + no postinstall before install |
</threat_model>

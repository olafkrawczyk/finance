<objective>
Add Better Auth authentication to the app and protect all Phase 1 routes. Configure Better Auth with a `pg` Pool (separate from the app's postgres.js client), enable `emailAndPassword` for testability, register Google + GitHub social providers, mount the handler at `/api/auth/*`, add a `requireAuth` middleware, and apply it to every Phase 1 domain route.

Purpose: REQ-6 requires OAuth/SSO. The import endpoints in later plans depend on a session being available; this plan establishes the auth backbone and the testable session pattern (email/password) the rest of the phase relies on.

Output: `src/auth.ts`, `src/interface-adapters/api/auth.ts`, modified `index.ts` and all three Phase 1 route files, `tests/auth.test.ts`, and a corrected `tests/api.test.ts` (auth-aware).
</objective>

|----------|-------------|
| Browser → /api/auth/* | OAuth redirect + callback; untrusted until session established |
| Browser → protected routes | Session cookie must be validated on every request |
| App → OAuth providers (Google/GitHub) | External token exchange |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-01 | Spoofing | Session cookie | mitigate | Better Auth issues HTTP-only, Secure, SameSite=Lax cookies; `requireAuth` validates via `auth.api.getSession` on every protected request (ASVS V3) |
| T-2-02 | Elevation of Privilege | Phase 1 routes | mitigate | `requireAuth` middleware on every ledger/opening-balance/reference route returns 401 without a valid session (ASVS V4) |
| T-2-03 | Spoofing | OAuth callback | mitigate | Better Auth built-in CSRF/state protection on OAuth callback (ASVS V2) |
| T-2-INFO | Information Disclosure | OAuth client secrets, BETTER_AUTH_SECRET | mitigate | All secrets read from env vars; never logged, never sent to client (ASVS V13) |

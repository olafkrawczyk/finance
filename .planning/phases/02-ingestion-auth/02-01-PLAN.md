<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Hono API | Untrusted OAuth callbacks and session cookies cross here |
| Hono API → Better Auth | Internal boundary; Better Auth manages its own pg Pool |
| Better Auth → PostgreSQL | Auth tables (users, sessions, accounts) live in same DB as app tables |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | OAuth callback | mitigate | Better Auth built-in CSRF protection for OAuth state parameter |
| T-02-02 | Tampering | Session cookie | mitigate | Better Auth uses HTTP-only, Secure, SameSite=Lax cookies |
| T-02-03 | Information Disclosure | OAuth client secrets | mitigate | Secrets read from env vars only; never logged or sent to client |
| T-02-04 | Elevation of Privilege | Protected routes | mitigate | `requireAuth` middleware on all Phase 1 routes; returns 401 if no session |
| T-02-05 | Denial of Service | Auth endpoint flooding | accept | Low-value target; rate limiting deferred to production hardening (Phase 5) |
</threat_model>

Purpose: This is the core processing engine of the import pipeline. Without the worker, CSV uploads would sit in the queue forever. The worker handles the complex format-specific parsing via LLM, batching for large historical imports, and atomic deduplication.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-14 | Tampering | LLM output format | mitigate | `ParsedTransactionSchema` validates every field before insert |
| T-02-15 | Tampering | LLM prompt injection | mitigate | Few-shot prompt has explicit formatting rules; no user-controlled text in system prompt |
| T-02-16 | Information Disclosure | OpenRouter API key | mitigate | Key read from `OPENROUTER_API_KEY` env var only; never logged |
| T-02-17 | Denial of Service | Infinite worker loop | mitigate | Visibility timeout + max retries (3) prevent stuck messages; archive on success, delete on max retries |
| T-02-18 | Repudiation | Lost job status | mitigate | `import_jobs` table tracks status transitions persistently |
| T-02-19 | Elevation of Privilege | Worker writes as any account | accept | Worker uses account_id from PGMQ payload; account_id was validated at enqueue time |
</threat_model>

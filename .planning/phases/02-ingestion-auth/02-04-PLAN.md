<objective>
Build the import worker: a PGMQ polling loop that dequeues import_queue jobs, preprocesses the CSV (CP1250 decode handled upstream; ING header detection / IPKO Blokada filtering here), calls OpenRouter with a few-shot JSON-schema prompt in batches of 50, validates LLM output with the Zod ParsedTransactionSchema, normalizes amounts, computes SHA-256 import_hash, inserts in atomic batches with ON CONFLICT DO NOTHING, updates import_jobs status/processed/errors, and archives or retries the message.

Purpose: REQ-4.2 (PGMQ worker + OpenRouter), REQ-4.3 (ING format), REQ-4.4 (IPKO format), REQ-4.6 (few-shot JSON prompt), REQ-4.7 (batch bulk import). Depends on 02-02 (import_jobs, import_queue, ParsedTransactionSchema, entities).

Output: `src/workers/import-worker.ts`, `tests/import-parse.test.ts`, `tests/import-worker.test.ts`, `tests/import-llm.test.ts`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PGMQ message → worker | Job payload (CSV content) dequeued and processed |
| Worker → OpenRouter (external) | Prompt sent; untrusted JSON returned |
| LLM JSON → transactions table | Untrusted parsed data inserted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-10 | Tampering | LLM output → DB insert | mitigate | ParsedTransactionSchema (Zod) validates every item; invalid items excluded; sql.begin rolls back failed batches (ASVS V5) |
| T-2-11 | Tampering | CSV preprocessing | mitigate | Header detection + Blokada filter operate on decoded content; metadata rows stripped before LLM; exact-count tests (83/252) guard regressions |
| T-2-12 | Spoofing/Tampering | OpenRouter prompt injection | mitigate | Few-shot prompt with explicit rules; output re-validated with Zod regardless of model claims (ASVS V5) |
| T-2-KEY | Information Disclosure | OPENROUTER_API_KEY | mitigate | Key read from env var only; never logged, never sent to client (ASVS V13) |
| T-2-DUP | Spoofing | duplicate processing on visibility timeout | mitigate | VISIBILITY_TIMEOUT=300s prevents premature re-read; import_hash dedup is the final gate (Pitfall 6) |
| T-2-SQLI | Tampering | worker DB inserts | mitigate | postgres.js tagged templates auto-parameterize all values (ASVS V5) |
| T-2-SC | Tampering | npm installs (none new) | accept | No new packages; crypto is Bun built-in; postgres already audited [OK] in RESEARCH |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /import | Untrusted file upload and account_id cross here |
| API → PGMQ | Enqueued job payload contains raw CSV content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Tampering | File upload path traversal | mitigate | Store file content in memory only (FormData File API); never use user-provided filename on filesystem |
| T-02-10 | Spoofing | Replay of import job | mitigate | `requireAuth` on /import; only authenticated users can enqueue jobs |
| T-02-11 | Tampering | Invalid account_id | mitigate | `z.uuid().safeParse()` validates account_id before enqueue |
| T-02-12 | Information Disclosure | CSV content in response | mitigate | Response contains only job_id, never the raw CSV content |
| T-02-13 | Denial of Service | Large CSV upload | accept | 10MB implicit limit via Bun; explicit limit can be added in Phase 5 |
</threat_model>

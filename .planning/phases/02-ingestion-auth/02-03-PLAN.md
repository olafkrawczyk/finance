<objective>
Build the authenticated import HTTP surface: `POST /import` accepts a multipart CSV upload + account_id, decodes the file as CP1250 (latin1) for BOTH ING and IPKO, auto-detects the bank format, and enqueues a PGMQ job atomically with an import_jobs row; `GET /import/:job_id` returns status. Also add the use-cases (enqueueImportJob, getImportStatus, detectFormat) and a dedup test proving the import_hash UNIQUE constraint silently skips duplicates.

Purpose: REQ-4.1 (async endpoint that enqueues and returns job_id immediately) and REQ-4.5 (dedup contract). Depends on 02-01 (requireAuth) and 02-02 (import_jobs table, import_queue, schemas).

Output: `src/core/import/use-cases.ts`, `src/interface-adapters/api/import.ts`, modified `index.ts`, `tests/import-api.test.ts`, `tests/import-dedup.test.ts`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → POST /import | Untrusted multipart upload (file + account_id) |
| App → PGMQ | Enqueue payload (CSV content) into queue |
| Browser → GET /import/:job_id | Untrusted job_id path param |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-07 | Elevation of Privilege | POST /import, GET /import/:job_id | mitigate | `requireAuth` on both routes; unauthenticated requests get 401 (ASVS V4) |
| T-2-08 | Tampering | account_id form field | mitigate | `ImportUploadSchema` (z.uuid) validates account_id; FK to accounts enforced at DB; 400 on invalid (ASVS V5) |
| T-2-09 | Tampering | duplicate/replayed import | mitigate | `import_hash` UNIQUE + ON CONFLICT DO NOTHING silently skips duplicates (REQ-4.5) |
| T-2-IDOR | Information Disclosure | GET /import/:job_id | accept | Single-user MVP; job ownership scoping deferred (no per-user account isolation yet). Documented as accepted risk for MVP. |
| T-2-FILE | Tampering | uploaded file path | mitigate | File read from in-memory buffer via arrayBuffer(); no user-provided filename touches the filesystem (ASVS V12) |
| T-2-SC | Tampering | npm installs (none new) | accept | No new packages; postgres/hono already audited [OK] in RESEARCH |

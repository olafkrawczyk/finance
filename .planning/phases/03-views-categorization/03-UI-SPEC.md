## Interaction Patterns

### Navigation
- **Routing:** Client-side `window.history.pushState` + `popstate` listener (extend existing pattern in `App.tsx`)
- **Header nav:** Horizontal button bar in `<header>`. Active route gets `bg-slate-900 text-blue-400`. Inactive gets `text-slate-400 hover:text-slate-200`.
- **Chart drill-down (D-03):** Click any chart data point → extract `activeLabel` (month string "YYYY-MM") → `navigateTo('/month/' + activeLabel)`
- **Post-import flow (D-10):** Import success screen → add "Categorize Transactions" button → navigates to `/categorize`

### Categorization UX (D-09)
- **Selection:** Checkbox per row on uncategorized transactions. Select-all checkbox in table header.
- **Assignment:** Single `CategoryDropdown` below/above table. "Save Categories" button triggers parallel `PATCH /transactions/:id/category` calls via `Promise.all`.
- **Feedback:** Loading state while saving ("Saving categories..."), success confirmation when all complete, error state with partial failure message if any PATCH fails.
- **Empty state:** When all transactions are categorized → "All caught up!" empty state.

### Manual Entry (D-11, D-12)
- **Form fields:** Category (dropdown), Amount (number input, step="0.01"), Description (text), Date (date input, default `new Date().toISOString().slice(0, 10)`), Type (select: income/expense/transfer)
- **Hidden field:** `account_id` — uses first account from `GET /accounts` (agent's discretion, per CONTEXT.md)
- **Amount conversion:** User input `"123.45"` → pad to `"123.4500"` before POST (match `NUMERIC(19,4)` format)
- **Validation:** HTML5 native (`required`, `type="number"`, `min="0.01"`). API returns Zod validation errors which are displayed inline.

### Data Loading States
- **Spinner:** `animate-spin rounded-full h-10 w-10 border-b-2 border-blue-500` (reuse existing ImportStatus pattern)
- **Skeleton:** Not used — spinner is sufficient for this phase.
- **Retry:** Error state includes "try again" messaging. No auto-retry.

### Responsive Behavior (D-08)
- **Breakpoints:** Tailwind defaults (`sm: 640px`, `md: 768px`, `lg: 1024px`, `xl: 1280px`). Target: iPhone (375px wide).
- **Tables:** `overflow-x-auto` wrapper with `min-w-[700px]` for horizontal scroll on mobile. Table headers remain visible.
- **Dashboard charts:** Stack vertically on mobile (`flex-col`), 2×2 grid on desktop. Each chart uses `ResponsiveContainer width="100%" height={300}`.
- **Header nav:** Compact labels, maybe icon-only or abbreviated on very small screens. `flex-wrap` for overflow.
- **Forms:** Full-width inputs on mobile, constrained `max-w-lg` on desktop.
- **Sidebar (Monthly view):** Below transaction table on mobile (`flex-col`), beside table on desktop (`lg:flex-row`).

## Data Normalization Contract

All API responses return `NUMERIC(19,4)` values as **strings**. Frontend MUST normalize before display:

```typescript
// Required normalization functions (create in api.ts or dedicated lib)
function normalizeSummaryRow(row: any): NormalizedSummaryRow {
  return {
    month: row.month,
    wydatki: parseFloat(row.wydatki),
    przychody: parseFloat(row.przychody),
    stan_konta: row.stan_konta != null ? parseFloat(row.stan_konta) : null,
    wydatki_bez_stalych: parseFloat(row.wydatki_bez_stalych),
    zaoszczedzone: parseFloat(row.zaoszczedzone),
    zaoszczedzone_log: parseFloat(row.zaoszczedzone_log),
  };
}

function normalizeTransaction(row: any): NormalizedTransaction {
  return {
    ...row,
    amount: parseFloat(row.amount),
  };
}
```

**Number formatting:** `new Intl.NumberFormat('pl-PL', { minimumFractionDigits: 2, maximumFractionDigits: 2 }).format(n)` for Polish locale display.

**Source:** RESEARCH.md Pitfall 1 — string-to-number conversion critical for Recharts rendering.

    - Build a `Map<string, Category>` from categories for O(1) lookup by `category_id`

    - Map each transaction: `parseFloat(t.amount)` for amount

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser→API (via api.ts functions) | All data fetching goes through api.ts which uses `credentials: 'include'` and unwraps the API envelope. Backend validates auth via `requireAuth`. |
| URL param → MonthlyPage | `yearMonth` prop comes from URL parsing in App.tsx (user-controlled). MonthlyPage splits on '-' and uses as date_from/date_to — if malformed, API returns 400/500 which is caught. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-12 | Information Disclosure | DashboardPage, ZbiorczyPage | accept | Data fetched via authenticated API calls. React auto-escapes JSX. No sensitive data exposed beyond what API returns. |
| T-03-13 | Tampering | MonthlyPage yearMonth parsing | mitigate | `yearMonth.split('-')` could produce NaN for malformed input. API validates date_from/date_to via Zod (`z.iso.date()`) — malformed dates return 400. Frontend catches and displays error. |
| T-03-14 | Denial of Service | DashboardPage linearRegression | accept | Pure client-side computation on <= 100 data points. O(n) algorithm. No network amplification. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan. |
</threat_model>

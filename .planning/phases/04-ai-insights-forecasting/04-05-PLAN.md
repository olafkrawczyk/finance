<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser→API | Frontend components fetch insights from /insights API. Session cookie sent via credentials: 'include'. API responses rendered as HTML — XSS surface if API returns unsanitized HTML. |
| User→UI | Dismiss action is destructive (permanent). Confirmation dialog prevents accidental dismissals. |
| User→UI | Generate button triggers costly LLM operations. Rate limiting is server-side; frontend shows cooldown feedback. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-23 | Information Disclosure | InsightsPage rendering another user's insights | mitigate | API requires auth (requireAuth). Frontend sends session cookie. If session is invalid/expired, API returns 401 and frontend shows error state. No data rendered. |
| T-04-24 | Tampering | XSS via insight.content field | mitigate | React's JSX auto-escapes string content by default (textContent, not innerHTML). Insight content is rendered as JSX text nodes, not dangerouslySetInnerHTML. Standard React protection. |
| T-04-25 | Repudiation | Accidental insight dismissal | mitigate | DismissConfirmDialog requires explicit user confirmation before calling dismissInsight. "Dismiss" button styled as destructive (red) to signal irreversibility. |
| T-04-26 | Denial of Service | Rapid Generate button clicks | mitigate | Frontend disables button during generation (generating state). Server-side rate limit (5 min cooldown) enforced in worker — even if frontend bypass is attempted. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages required. React, Recharts, Tailwind already installed. |
</threat_model>

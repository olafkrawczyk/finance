    - id: UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - user_id: TEXT NOT NULL REFERENCES "user"(id) ON DELETE CASCADE
    - type: TEXT NOT NULL CHECK (type IN ('alert', 'tip', 'trend', 'forecast'))
    - priority: TEXT NOT NULL CHECK (priority IN ('high', 'medium', 'low'))
    - title: TEXT NOT NULL
    - content: TEXT NOT NULL
    - linked_transaction_ids: UUID[] NOT NULL DEFAULT '{}' (per D-04)
    - linked_category_ids: UUID[] NOT NULL DEFAULT '{}' (per D-04)
    - dismissed: BOOLEAN NOT NULL DEFAULT false (per D-04: dismissed insights don't reappear)
    - dedup_hash: TEXT NOT NULL (per D-14: SHA-256 of type|title|content)
    - created_at: TIMESTAMPTZ NOT NULL DEFAULT now()

    Indexes (per RESEARCH.md pattern recommendations):
    - idx_insights_user_type ON insights(user_id, type)
    - idx_insights_user_dismiss ON insights(user_id, dismissed)
    - idx_insights_created_at ON insights(created_at DESC)
    - idx_insights_dedup ON insights(dedup_hash, created_at)

    Immutability trigger: CREATE OR REPLACE FUNCTION block_insight_immutable_change() — only allows changing the dismissed flag. <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DB→Application | Insights table stores LLM-generated content; dedup_hash is SHA-256 (non-reversible), but content field may contain LLM output including hallucinated financial claims |
| Schema validation boundary | Zod schemas are the only gate between raw LLM JSON output and the database; malformed or malicious JSON (including prompt injection remnants in content) must be caught by safeParse filtering |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | insights table UPDATE (immutability bypass) | mitigate | block_insight_immutable_change trigger enforces only dismissed flag can change; all other fields must match OLD values. DB-level constraint prevents accidental/modified content insertion. |
| T-04-02 | Information Disclosure | dedup_hash column | accept | SHA-256 hash is one-way and reveals no transaction data. Accept risk. |
| T-04-03 | Elevation of Privilege | insights table — user_id foreign key | mitigate | FOREIGN KEY REFERENCES "user"(id) ON DELETE CASCADE ensures only valid users own insights. User deletion cascades to remove their insights. |
| T-04-04 | Denial of Service | insights table — array columns (linked_transaction_ids, linked_category_ids) | accept | PostgreSQL array columns are bounded by individual row size limits. No vector for unbounded growth faster than normal usage. Accept risk. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages required for this plan. All dependencies already installed and vetted in prior phases. |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PGMQ→Worker | Queue messages contain { user_id, triggered_by } — user_id drives data scope but is generated server-side (trusted). triggered_by discriminates rate limit enforcement. |
| Worker→OpenRouter | Raw transaction data sent to Claude Sonnet 4 (trusted per Anthropic no-training policy). Anonymized CategoryAggregate[] sent to DeepSeek R1 (free tier — untrusted). The code boundary between these two data paths MUST be strict. |
| OpenRouter→Worker | LLM responses are untrusted — JSON parsed, Zod validated, invalid items filtered. Content stored in insights.content may contain hallucinated financial claims. |
| Worker→DB | Insights inserted into insights table — dedup_hash prevents exact duplicates. User-scoped via user_id foreign key. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Spoofing | OpenRouter API response | mitigate | Response validated: content extracted from choices[0].message.content, JSON.parsed, then Zod safeParse per item. Malformed or spoofed responses filtered to empty arrays. |
| T-04-06 | Tampering | Prompt injection via transaction descriptions | mitigate | sanitizePromptText() strips non-printable characters (\x00-\x08, \x0B-\x0C, \x0E-\x1F, \x7F) from all transaction descriptions before prompt assembly. Control characters that could confuse LLM parsing are removed. |
| T-04-07 | Information Disclosure | DeepSeek R1 prompt contains raw transaction data | mitigate | TypeScript type signature: callDeepSeekForForecast(aggregates: CategoryAggregate[]) — compiler prevents passing TransactionData[]. buildForecastPrompt() receives only numerical aggregates. Test in insights-worker.test.ts verifies R1 prompt string contains zero transaction descriptions (grep assertion). |
| T-04-08 | Information Disclosure | API key in error messages | mitigate | OpenRouter error handling: throw Error(`OpenRouter error: ${response.status} ${errorText}`) — includes status code only. Never include OPENROUTER_API_KEY, raw response body, or request payload in thrown errors (following import-worker pattern at line 179). |
| T-04-09 | Repudiation | LLM hallucination presenting false financial advice | mitigate | Per D-04: every insight links to linked_transaction_ids as evidence. Users can verify claims against their own data. Insight card UI shows "See related transactions" link. |
| T-04-10 | Denial of Service | Rate limit abuse via rapid on-demand generation | mitigate | RATE_LIMIT_COOLDOWN_MS = 5 minutes (300000ms). Manual triggered_by='manual' messages are skipped if last manual generation was within cooldown window. Cooldown tracked via most recent insight created_at for that user. |
| T-04-11 | Denial of Service | Free-tier R1 rate limit (429) causing worker failures | mitigate | callDeepSeekForForecast() catches non-ok responses, logs warning, returns [] (empty array). Claude insights still generated. Graceful degradation: narrative insights work without numerical forecasts. |
| T-04-12 | Elevation of Privilege | Worker processes messages for non-existent users | mitigate | getInsightDataWindow(userId) query will return empty transaction set for invalid userId. No data leak. Worker archives message after processing (even with empty result set). |
| T-04-13 | Information Disclosure | Insight dedup_hash leaks via timing side-channel | accept | SHA-256 hash is one-way. Timing differences in SELECT ... WHERE dedup_hash = ANY() are uniform across hash values. Accept risk. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages required for this plan. All dependencies (crypto via Bun built-in, fetch via Bun built-in, postgres, zod) already installed and vetted in prior phases. |
</threat_model>

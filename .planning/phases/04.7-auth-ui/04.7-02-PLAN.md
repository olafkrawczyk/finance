    **frontend/src/api.ts** — Add a helper function that wraps fetch and redirects to /login on 401:

    1. Add this utility function at the top of the file (or replace with a TDD refactor approach — if the file is large, add the helper at the top and update `redirectToLogin` in existing functions):

    ```typescript
    function redirectToLogin() {
      if (window.location.pathname !== '/login') {
        window.location.pathname = '/login';
      }
    }
    ```

    2. Each existing API function that calls `fetch()` should check for 401 and redirect. The cleanest approach: create an internal `apiFetch` wrapper:

    ```typescript
    async function apiFetch(input: RequestInfo | URL, init?: RequestInit): Promise<Response> {
      const res = await fetch(input, { credentials: 'include', ...init });
      if (res.status === 401) {
        redirectToLogin();
        throw new Error('Session expired — redirecting to login');
      }
      return res;
    }
    ```

    3. Update all existing API functions to use `apiFetch` instead of raw `fetch()`. For each function:
       - Replace `fetch(...)` with `apiFetch(...)`
       - Remove `credentials: 'include'` from the options (apiFetch adds it)
       - Keep all existing error handling — the 401 check fires before other error checks

    This ensures that if the session expires mid-session (e.g. server restart, token expiry), any subsequent API call triggers a redirect to /login rather than showing a raw "Failed to fetch X: Unauthorized" error.
  </action>

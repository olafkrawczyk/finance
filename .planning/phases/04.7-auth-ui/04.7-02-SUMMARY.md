## Accomplishments
- Added `authClient.useSession()` to App.tsx for reactive session state — three render modes: loading spinner (isPending), login page (no session, no header/footer), full authenticated app
- Added `/login` route in the inline router for direct navigation and OAuth callback redirects
- Added user email display (`session.user.email`) and "Wyloguj" logout button right-aligned in header after navigation section
- Added `redirectToLogin()` and `apiFetch()` in api.ts — all 15 existing API functions migrated from raw `fetch()` to `apiFetch()` which checks for 401 and redirects to `/login` on session expiry
- Removed redundant `credentials: 'include'` from individual API calls (apiFetch adds it centrally)

- `frontend/src/api.ts` — Added `redirectToLogin()` and `apiFetch()` wrapper; all functions migrated to use `apiFetch`

## Decisions Made
- **`useSession()` over manual fetch:** Better Auth's reactive hook handles session lifecycle, caching, and auto-updates — no manual state management needed
- **Full reload on 401:** `window.location.pathname = '/login'` causes a full page navigation on session expiry, ensuring clean application state
- **Logout triggers reactive re-render:** After `authClient.signOut()`, the session atom updates automatically, causing App.tsx to re-render to the login page — no manual redirection needed
- **apiFetch centralizes credentials and 401 check:** All 15 existing API functions now use the wrapper, removing 32 redundant `credentials: 'include'` lines

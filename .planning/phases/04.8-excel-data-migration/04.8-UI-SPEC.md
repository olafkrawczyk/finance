| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 | 1.43 | `text-sm` |
| Label | 14px | 600 | 1.43 | `text-sm font-semibold` |
| Heading | 24px | 600 | 1.33 | `text-2xl font-semibold` |

**Text Colors:**
- Primary text: `text-slate-100`
- Secondary text: `text-slate-200`
- Muted text: `text-slate-400`
- Alert/Warning text: `text-red-400`

## Color

A strict 60/30/10 split layout with one destructive semantic color scheme.

| Role | Value | Tailwind Class | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `#020617` | `bg-slate-950` | Page background (`min-h-screen`) |
| Secondary (30%) | `#0f172a` | `bg-slate-900` | Cards, Confirmation Modals, Dialogs, active nav background overlays |
| Accent (10%) | blue-600 → indigo-600 | `from-blue-600 to-indigo-600` | Primary action buttons, gradient progress bar fill, focus rings |
| Destructive | red-950 background / red-800 border | `bg-red-950/50 border border-red-800` | Warning modal panels, delete/wipe confirm button, error state displays |

- **Accent color** is reserved strictly for primary buttons, focus state borders, active routes, and progress indication.
- **Card borders** must use `border border-slate-800` or `border-slate-800/80`.
- **Text on buttons:** Primary action text must be `text-white` over the gradient. Secondary/cancel button text must be `text-slate-300` on `bg-slate-900`.

## Interaction Patterns & States

### 1. Route Accessibility
- The `/migration` path must be typed manually into the URL bar to be accessed.
- It is excluded from the navigation bar in `App.tsx` to prevent accidental clicks by users during normal app operations.
- Direct root checks redirect unauthenticated sessions to `/login`.

### 2. Upload Dropzone States
- **Default/Idle State:** Dashed gray border (`border-slate-800`). Instructs the user to drag and drop or browse. Primary button "Start Ingestion" remains disabled (`opacity-50 cursor-not-allowed`).
- **Drag-Over State:** Dashed border transitions to active blue/indigo gradient styling (`border-blue-500`).
- **File Selected State:** Dashed border returns to normal but displays the file details (`name: budget.xlsx`, size, type) and a "Remove File" trash icon. The "Start Ingestion" primary action button becomes active.
- **File Rejection State:** If a user drops a non-`.xlsx` file (e.g. a CSV or PDF), the border turns red (`border-red-800`), an inline validation error shows "Only .xlsx Excel workbooks are supported", and the selection is rejected.

### 3. Modal Confirmation flow
- Clicking "Start Ingestion" displays the confirmation modal backdrop overlay (`fixed inset-0 bg-slate-950/80 backdrop-blur-sm z-50 flex items-center justify-center p-4`).
- Inside the modal, the destructive card contains:
  - Destructive warning banner with red icon and background (`bg-red-950/50 border border-red-800 text-slate-100 rounded-xl p-6`).
  - Text description detailing the database truncate execution (CASCADE).
  - Safety validation challenge input: User must type **`MIGRATE`** exactly.
  - The "Yes, Wipe Data and Migrate" action button remains disabled until the input exactly matches `MIGRATE`.
  - Clicking "Cancel" closes the modal safely, resetting the text input.

### 4. Progress Polling Layout & Logic
- Upon confirmation, the modal closes and the page transitions to a non-interactive Progress Tracking screen.
- A request is made to `POST /api/migration/excel`. Upon receiving a 202 status and a `{ job_id }`, polling starts immediately.
- **Polling Loop:**
  - Interval: `2000ms`.
  - Endpoint: `GET /api/migration/excel/:jobId`.
- **UI Indicators during processing:**
  - A circular spinner (`animate-spin`) accompanied by the text: "Ingesting Excel Data...".
  - A horizontal progress bar reflecting `(processed_sheets / total_sheets) * 100` using the `from-blue-600 to-indigo-600` gradient fill.
  - Live details label: "Processing Sheet: {sheetName} ({processedSheets} of {totalSheets})".
- **Termination:**
  - If status resolves to `completed`: Polling stops. A success card is shown with a green checkmark, complete message, and a primary CTA "Go to Dashboard" button which redirects the page.
  - If status resolves to `failed` or throws a connection error: Polling stops. An error card is shown with a red alert icon, the error description, and a "Try Again" button which resets the page back to the default dropzone upload state.

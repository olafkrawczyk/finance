<objective>
**Backend Excel Ingestion Service**

This plan delivers the complete backend infrastructure to parse, map, route, and ingest historical financial transaction and opening balance data from an Excel workbook (`budget.xlsx`) spanning July 2020 to October 2025. It guarantees a clean ledger setup by executing a destructive database reset before queuing an asynchronous worker job to process the workbook sheets.

Key Outputs:
- A new Hono API router for `/api/migration/excel`.
- A timezone-safe Excel parsing engine supporting legacy and modern monthly sheets.
- Category translation rules mapping old categories (`dentysta`, `mpk`, `kawka`) to seeded ones, with fallback to `fun`.
- Account routing logic dispatching transactions to ING Business (`Konto Direct dla Firmy`) or PKO Personal (`IPKO`) based on rules.
- Queue integration processing `excel_migration` tasks and updating job status dynamically in `import_jobs`.
</objective>

  <read_first>src/infrastructure/db/seed.sql</read_first>

  <action>
    Create `src/core/import/excel-parser.ts` to implement the workbook extraction logic using `xlsx`:
    
    1. **Text Normalization Helper**:
       ```typescript
       export function normalizeText(str: string): string {
         return str
           .toLowerCase()
           .normalize('NFD')
           .replace(/[\u0300-\u036f]/g, '')
           .replace(/ł/g, 'l')
           .replace(/Ł/g, 'l')
           .trim();
       }
       ```
    2. **Sheet Chronological Sorter**:
       - Extract year and month number (1-12) from month names (styczen, luty, marzec, kwiecien, maj, czerwiec, lipiec, sierpien, wrzesien, pazdziernik, listopad, grudzien).
       - Skip helper sheets `zbiorczy`, `kategorie`, or any sheet that does not match a month.
       - If the sheet name does not contain a year suffix (legacy sheets Jul 2020 - Jan 2021):
         - Map `lipiec` to `grudzien` ➔ Year `2020`
         - Map `styczen` ➔ Year `2021`
       - Sort sheets: `(a, b) => (a.year - b.year) || (a.month - b.month)`.
    3. **Excel Date Parser**:
       - Safely convert cell values. If a number (serial date), use `XLSX.SSF.parse_date_code(val)` to extract `{y, m, d}` and construct `YYYY-MM-DD`. If a string matching a date regex (YYYY-MM-DD or DD.MM.YYYY), parse and normalize to `YYYY-MM-DD`. Default to the 1st of the month.
    4. **Opening Balance Cell Selector**:
       - Row index 2 (Row 3).
       - July 2020 – October 2020 (Year 2020, Month <= 10) ➔ Column index 6 (Col G).
       - November 2020 onwards ➔ Column index 7 (Col H).
    5. **Category Resolver**:
       - Look up categories fetched from the database.
       - Normalize legacy spreadsheet names: `dentysta` ➔ `lekarz`, `mpk` ➔ `przejazdy`, `kawka` ➔ `kawa`.
       - Check if the normalized category matches a normalized database category. If found, return database category name & ID. If not, default to `fun` category.
    6. **Account Router**:
       - Route to **ING Business** (`Konto Direct dla Firmy`) if category is `VAT`, `ZUS`, `PIT36` (or contains `PIT`), or `paliwo`.
       - Route to **ING Business** if category is `auto` AND amount > 2000.00 PLN.
       - Route to **ING Business** if description contains `PPE`, `ORANGE`, or `PLAY` (case-insensitive).
       - Route all others to **PKO Personal** (`IPKO`).
    7. **Transaction Parser**:
       - Modern Sheets (Feb 2021 onwards):
         - Expenses: Cols A–D (`kategoria`, `kwota`, `opis`, `data`). Skip if Col B (Amount) is empty or not a number. Skip header rows.
         - Incomes: Cols E–F (`nazwa`, `kwota`). Default date to 1st of the month. Category is NULL.
       - Legacy Sheets (Jul 2020 – Jan 2021):
         - Expenses: Cols A–B (`kategoria`, `kwota`). Description is NULL. Date defaults to 1st of the month.
         - Incomes: Jul-Oct 2020 ➔ Cols D–E; Nov 2020 - Jan 2021 ➔ Cols E–F. Default date to 1st of the month. Category is NULL.
       - Filter out columns R–U (ignore columns with index >= 17).
       - Build `import_hash` using SHA-256 of `${sheetName}|${rowNumber}|${date}|${amount}|${description}`.
  </action>

<threat_model>
## Trust Boundaries
- **client → API**: Users upload files and trigger destructive DB wipes via POST requests. Auth checks prevent unauthorized resets.
- **API → worker**: Uploaded file path passed via queue message must remain secure.

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-MIG-01 | Elevation of Privilege | POST /api/migration/excel | mitigate | Enforce Better Auth session validation at route entry. Reject unauthenticated requests with 401. |
| T-MIG-02 | Denial of Service | File Upload Limit | mitigate | Enforce 20MB file size limit in multipart parser. Asynchronously stream file upload to disk to protect server memory. |
| T-MIG-03 | Tampering | SQL Injection | mitigate | Use pg-safe parameterized template strings (`sql` queries) for all truncations, queries, and batch inserts. |
| T-MIG-04 | Tampering | Path Traversal / Arbitrary File Write | mitigate | Generate job IDs as V4 random UUIDs on the server side. Do not trust file names or client-provided IDs. Save files strictly in `scratch/uploads/`. |
| T-MIG-05 | Information Disclosure | Temp file bloat | mitigate | In workers, ensure the workbook file is deleted in a `finally` block, ensuring deletion even on failures. |
</threat_model>

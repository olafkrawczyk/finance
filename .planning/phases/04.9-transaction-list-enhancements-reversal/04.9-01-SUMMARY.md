## Decisions Made

- **Immutable reversal via .toReversed():** The initial implementation used `.reverse()` (mutating). A subsequent refactor in Plan 03 changed it to `.toReversed()` for explicit immutability, which is semantically clearer and avoids side effects if the array is referenced elsewhere.
- **Chronological-first computation:** The SQL aggregation still uses `ORDER BY month ASC` so that running balance propagation is correct. The reversal happens only at the return boundary, keeping the computation logic simple and testable.

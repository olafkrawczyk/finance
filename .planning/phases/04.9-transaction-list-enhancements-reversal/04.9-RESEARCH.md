# Phase 04.9: Transaction List Enhancements & Reversal - Research

This document outlines the domain analysis, implementation approach, technical details, dependencies, and validation architecture for Phase 04.9: Transaction List Enhancements & Reversal.

---

## 1. Domain Analysis

### Phase Boundary
The goal of this phase is to deliver two high-priority user interface and backend usability improvements:
1. **Transaction List Enhancements (`MonthlyPage.tsx`)**: In-memory searching, sorting, and filtering of transactions for a specific month. Since the page fetches up to 500 transactions (page-size capped), client-side filtering offers a latency-free experience.
2. **Reversal of Monthly Summaries (`ZbiorczyPage.tsx`)**: Reversing the order of monthly summaries so the newest month appears first (descending order) while keeping the chronological balance calculations mathematically correct.

---

## 2. Implementation Approach

### Backend Reversal Logic
- **Target Use Case**: `getMonthlySummary()` in `src/core/ledger/use-cases.ts`.
- **Logic Flow**:
  1. Retrieve aggregate counts and opening balances ordered chronologically (ascending) via SQL.
  2. Compute running balance (`stan_konta`) sequentially from oldest to newest month.
  3. Reverse the final array using `.reverse()` before returning it to the handler.
  This satisfies **D-04** by keeping calculations correct but outputting the results newest-first.

### Frontend Transaction Filter UI & Logic
- **Component**: `frontend/src/pages/MonthlyPage.tsx`.
- **State Variables**:
  - `searchTerm` (string, default `''`)
  - `selectedType` (string, default `'all'`)
  - `selectedCategory` (string, default `'all'`)
  - `sortBy` (string, default `'date-desc'`)
- **Filter Hook (`useMemo`)**:
  - Performs case-insensitive matching on `description` and `category_name` (**D-03**).
  - Filters by type (`expense`, `income`, `transfer`).
  - Filters by category (dynamically populated from unique category names present in the month's transaction list).
  - Sorts by date (newest/oldest first) and amount (highest/lowest first), or description (A to Z) (**D-02**).
- **Filter Grid Panel**:
  - Always visible grid layout placed above the transaction table (**D-01**).
  - Matches the design system colors: Slate background, active borders, and focus rings (**UI-SPEC**).
  - Reset button to clear all inputs to defaults.
  - Visible count badge: showing `N` of `M` matching transactions.
- **Empty State Display**:
  - Renders a clean card when no transactions match the active filters, using copy specified in the contract.

## 3. Technical Details

### Search & Filter Logic Details
```typescript
const filteredTransactions = useMemo(() => {
  if (!transactions) return [];
  
  return transactions
    .filter((tx) => {
      // 1. Search text match (case-insensitive)
      const matchesSearch =
        !searchTerm ||
        (tx.description ?? '').toLowerCase().includes(searchTerm.toLowerCase()) ||
        (tx.category_name ?? '').toLowerCase().includes(searchTerm.toLowerCase());
        
      // 2. Type match
      const matchesType =
        selectedType === 'all' ||
        tx.type === selectedType;
        
      // 3. Category match
      const matchesCategory =
        selectedCategory === 'all' ||
        tx.category_name === selectedCategory;
        
      return matchesSearch && matchesType && matchesCategory;
    })
    .sort((a, b) => {
      // 4. Sort selection
      switch (sortBy) {
        case 'date-desc':
          return new Date(b.date).getTime() - new Date(a.date).getTime() || b.id.localeCompare(a.id);
        case 'date-asc':
          return new Date(a.date).getTime() - new Date(b.date).getTime() || a.id.localeCompare(b.id);
        case 'amount-desc':
          return b.amount - a.amount;
        case 'amount-asc':
          return a.amount - b.amount;
        case 'description-asc':
          return (a.description ?? '').localeCompare(b.description ?? '');
        default:
          return 0;
      }
    });
}, [transactions, searchTerm, selectedType, selectedCategory, sortBy]);
```

### Unique Category Extraction
```typescript
const uniqueCategories = useMemo(() => {
  if (!transactions) return [];
  const cats = new Set<string>();
  transactions.forEach((tx) => {
    if (tx.category_name) cats.add(tx.category_name);
  });
  return Array.from(cats).sort();
}, [transactions]);
```

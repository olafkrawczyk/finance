    Addresses review concern: LOW severity `.reverse()` mutability (Reviews P3 — OpenCode suggestion).

    In `src/core/ledger/use-cases.ts`, line 112, replace `.reverse()` with `.toReversed()`.

    The current code at line 87-112 chains `.reverse()` after `.map()`:
    ```
    return agg.map((row: ...) => { ... return { ... }; }).reverse();
    ```
    
    The `.map()` creates a new array, so `.reverse()` mutating it in place is technically safe — the original `agg` array is not affected. However, `.toReversed()` (ES2023) returns a new reversed copy without mutating the source, making the immutability contract explicit and preventing future refactors from accidentally sharing the array reference.

    Check whether Bun supports `.toReversed()` by running `bun -e "console.log(typeof [].toReversed)"`. If it returns `"function"`, make the replacement. If Bun raises a ReferenceError, leave `.reverse()` unchanged and add a comment documenting the choice:
    ```
    // Using .reverse() because .toReversed() is not available in this Bun version.
    // .map() creates a new array so mutability is not a concern.
    ```
    Do NOT use `npx` or install any packages to test this — just `bun -e`.
  </action>

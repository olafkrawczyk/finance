  <action>
    Add filtering, sorting, and searching states and logic to `frontend/src/pages/MonthlyPage.tsx`.

    1. **Define State Variables**:
       - `searchTerm` (string, default `''`)
       - `selectedType` (string, default `'all'`)
       - `selectedCategory` (string, default `'all'`)
       - `sortBy` (string, default `'date-desc'`)

    2. **Populate Unique Categories**:
       - Add a `useMemo` hook to extract all unique, non-null `category_name` values from `transactions` and sort them alphabetically.

    3. **Implement useMemo Filter & Sort**:
       - Filter transactions based on `searchTerm` (case-insensitive description and category_name matching), `selectedType`, and `selectedCategory`.
       - Sort transactions based on `sortBy` (`date-desc`, `date-asc`, `amount-desc`, `amount-asc`, `description-asc`).
       - Handle null descriptions or categories gracefully without crashing.

    4. **Design and Insert Filter Panel UI**:
       - Insert a panel labeled with "Filtruj Transakcje" (CTA/Header) directly above the transaction table.
       - Use a grid layout (`grid grid-cols-1 md:grid-cols-4 gap-4`) for:
         - Search input: placeholder `"Szukaj w opisie lub kategorii..."` with a search icon prefix.
         - Dropdown for Type: `"Wszystkie typy"`, `"Przychody"`, `"Wydatki"`, `"Przelewy"`.
         - Dropdown for Category: `"Wszystkie kategorie"` + options from unique categories.
         - Dropdown for Sort: `"Data: Najnowsze najpierw"`, `"Data: Najstarsze najpierw"`, `"Kwota: Od najwyższej"`, `"Kwota: Od najniższej"`, `"Opis: A do Z"`.
       - Add a "Reset" or "Wyczyść" button next to/under controls to clear all inputs to defaults.
       - Add a dynamic counter displaying the visible vs total counts: e.g. `"Pokazuje N z M transakcji"`.
       - Style components using Slate colors matching `04.9-UI-SPEC.md`.

    5. **Render Empty & Error States**:
       - When filtered results array is empty, render the empty state card with heading `"Brak pasujących transakcji"` and body `"Spróbuj zmienić filtry lub wyszukiwaną frazę."`.
       - If there is a fetch error, render the error card with text `"Failed to load data — check connection and try again."` (overwriting/updating the default error text).
  </action>

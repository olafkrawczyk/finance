## Decisions Made

- **Cross-consumer ordering at component boundary rather than API layer:** DashboardPage now re-reverses the newest-first API data to chronological order. ZbiorczyPage continues using the newest-first response. This avoids the shared-consumer regression that would occur from changing the API contract.
- **`.toReversed()` over `.reverse()`:** Bun (v1.3.14) supports ES2023 `.toReversed()`. Used for explicit immutability — returns a new reversed copy without mutating the source array.
- **Inline pattern preserved:** The `.map().reverse()` pattern is kept inline rather than extracted to a separate function, maintaining consistency with existing code style.

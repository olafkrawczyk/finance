## Decisions Made

- **Pure function extraction strategy:** Filter/sort/category logic extracted to exported module-level functions rather than keeping them inside `useMemo` closures. This enables direct unit testing (import → call → assert) without needing React rendering infrastructure.
- **ID-based tiebreaker:** Using string ID comparison (`b.id.localeCompare(a.id)`) for same-date transactions. IDs are UUIDs with no inherent ordering semantics, but any deterministic ordering is sufficient for a tiebreaker — the guarantee is that sort order is reproducible, not that it follows any particular pattern.
- **Uncategorized filter value:** Used magic string `"uncategorized"` as the `selectedCategory` value rather than `null` or a sentinel. The filter logic checks `t.category_name === null` when this value is selected.
- **Polish locale parameter:** The `'pl'` locale is passed as the second argument to `localeCompare`. This is a standards-compliant approach that handles Polish diacritics (`ą`, `ć`, `ę`, `ł`, etc.) correctly in sorting.

  <action>
    Create `entrypoint.sh`:

    ```bash
    #!/bin/bash
    set -e

    cleanup() {
      echo "Shutting down..."
      kill -TERM $(jobs -p) 2>/dev/null || true
      wait
      echo "All processes exited."
    }
    trap cleanup SIGTERM SIGINT

    # Load .env.production if present (D-10: bun --env-file is used, but shell also needs vars)
    if [ -f /app/.env.production ]; then
      echo "Loading .env.production..."
      set -a
      source /app/.env.production
      set +a
    fi

    # Run database migrations (D-07)
    echo "=== Running database migrations ==="
    bun run src/infrastructure/db/migrate.ts
    echo "=== Migrations complete ==="

    # Start all 4 processes (D-01)
    PORT=${PORT:-3000}
    echo "=== Starting API server (port $PORT) ==="
    bun index.ts &

    echo "=== Starting insights worker ==="
    bun src/workers/insights-worker.ts &

    echo "=== Starting import worker ==="
    bun src/workers/import-worker.ts &

    # Wait for any process to exit
    echo "=== All processes started ==="
    wait
    ```

    Use `set -e` at the top so migration failure exits the container. Use `set -a`/`set +a` when sourcing .env.production to export all vars. The trap forwards SIGTERM/SIGINT to child processes for graceful shutdown.

## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host filesystem → Container | `.env.production` volume-mounted from host into container — secrets cross this boundary |
| Container → Internet | App exposed on port 3000 (though behind Cloudflare Tunnel in production) |
| Container → Postgres | App connects to db service via internal docker network |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | Dockerfile layer caching | mitigate | `.env.production` listed in `.dockerignore` — never copied into image layers |
| T-05-02 | Information Disclosure | `.env.production` volume mount | mitigate | File mounted `:ro` (read-only) inside container at `/app/.env.production` |
| T-05-03 | Tampering | Container image build | accept | Developer builds locally on trusted machine; no CI/CD pipeline in this phase |
| T-05-04 | Elevation of Privilege | Container runs as root | accept | Single-user homelab app; root in container is acceptable risk. Future: add USER directive |
| T-05-05 | Tampering | entrypoint.sh | accept | Controlled source file, committed to git, not modifiable at runtime |
| T-05-SC | Tampering | npm/bun package installs | mitigate | All packages from locked lockfile (`bun.lock`); node-pg-migrate verified via slopcheck (disposition: [OK]) |

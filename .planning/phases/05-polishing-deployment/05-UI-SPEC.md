# Phase 5 — UI Design Contract

> Visual and interaction contract for infrastructure/deployment phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Statement

**This phase contains zero new UI components, pages, or visual features.** Phase 5 is exclusively infrastructure and deployment:

1. **Dockerization** — Multi-stage Docker build with single-entry orchestration for 4 processes (API server, insights worker, import worker, static file serving via Hono).
2. **DB migration tooling** — node-pg-migrate integration, run on container start.
3. **Secrets management** — `.env.production` volume-mounted into container.
4. **Auth hardening** — Verify existing auth guard, session handling, and 401 redirects work correctly in production.

The only UI-related surface area is:
- Hono's `serveStatic` middleware serving the Vite-built `frontend/dist/` directory in production.
- Ensuring the login page, auth guard loading spinner, and 401 redirect loop function when served from the Docker container.

All visual/design contracts from Phase 04.9 remain in effect unchanged.

**Source:** CONTEXT.md D-01 through D-11, ROADMAP.md Phase 5 goal. [CONTEXT.md + ROADMAP.md]

## Production Serving Contract

### Hono Static File Serving

| Property | Value |
|----------|-------|
| Middleware | Hono `serveStatic` (`hono/serve-static`) |
| Built frontend location | `frontend/dist/` |
| Root path served | `/` |
| Fallback | Serve `index.html` for SPA client-side routing (all non-API routes) |
| API prefix | `/api/*` — must NOT be caught by static fallback |
| Order | Static middleware registered AFTER API routes and before catch-all fallback |

### Auth Flow in Production

| Flow | Expected Behavior | Production Concern |
|------|-------------------|--------------------|
| No session → any page | Redirect to `/login` | Must work with SPA routing from static build |
| Valid session → any page | Show full app layout | Session cookie must be sent with API requests |
| Login success | Redirect to `/dashboard` | Must work when served from Hono, not Vite dev proxy |
| Logout | Clear session, redirect to `/login` | Full page reload may be needed for static serving |
| Expired session | API returns 401 → global handler redirects to `/login` | Must trigger from served React build |
| Google OAuth callback | Better Auth handles callback → redirect to `/dashboard` | Must use production `BETTER_AUTH_URL` |

### CORS Configuration in Production

| Property | Value |
|----------|-------|
| Origin | `process.env.FRONTEND_URL` — must match production domain |
| Credentials | `true` — session cookies require credentials |
| Allowed headers | `Content-Type`, `Authorization` |
| Allowed methods | `POST`, `GET`, `OPTIONS` |

**Note:** In production, the frontend is served from the same origin as the API (same Hono server on same port). CORS should be effectively a no-op for same-origin requests, but the middleware must still allow auth route passthrough.

**Source:** `index.ts` lines 17-25, CONTEXT.md D-01/D-03. [Codebase analysis + CONTEXT.md]

## Interaction States: Auth Flow Diagram

```
┌─────────────────────────────────────────────┐
│  User visits any URL                          │
│  └─ isPending? → Show loading spinner         │
│     └─ No session? → Show LoginPage           │
│        ├─ Login tab → email/password form     │
│        │  └─ Submit → loading → redirect       │
│        ├─ Register tab → name+email/password  │
│        │  └─ Submit → loading → redirect       │
│        └─ Google button → OAuth redirect       │
│     └─ Has session? → Show full app layout     │
│        └─ Wyloguj → signOut → redirect         │
│        └─ Expired session → 401 → signOut →    │
│           redirect to /login                   │
└─────────────────────────────────────────────┘
```

All transitions must work identically whether served via Vite dev proxy (`localhost:5173`) or via Hono static serving from Docker.
